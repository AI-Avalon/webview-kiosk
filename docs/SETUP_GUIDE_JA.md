# Webview Kiosk - セットアップガイド（日本語）

## 概要

このドキュメントは、GalaxyTab A9+ などの受付端末にて、Webview Kioskを使用して、特定のウェブサイトへの制限されたアクセスを提供する方法を説明します。

本ガイドで説明する構成により、以下が実現されます：

- ✅ 指定のウェブサイト（例：https://www.e-shisetsu.e-aichi.jp/web_info.html）のみアクセス可能
- ✅ ユーザーがアプリケーションから外に出られない（Lock Task モード）
- ✅ 10分間の無操作でセッション自動クリア＆リセット
- ✅ 画面スリープ後の自動復帰
- ✅ 物理ボタン（戻る、メニュー等）の無効化
- ✅ 受付スタッフのみがパスワードで設定変更可能

## インストール手順

### Step 1: ファイル準備

1. **リリースAPK のダウンロード**
   - `webview-kiosk-v0.26.9-release.apk` をダウンロード

2. **デバイスへの転送**
   ```bash
   adb install webview-kiosk-v0.26.9-release.apk
   ```

### Step 2: デバイス設定（初期セットアップ）

#### デバイス所有者モードの設定

GalaxyTab A9+ では、以下の手順でデバイス所有者権限を付与します：

**重要：この作業は工場出荷状態またはデバイスをリセット後に行う必要があります。**

1. **デバイスをリセット**
   - 設定 → 一般管理 → リセット → リセット

2. **初期セットアップ中にコマンド実行**
   ```bash
   adb shell dpm set-device-owner uk.nktnet.webviewkiosk/.WebviewKioskAdminReceiver
   ```

3. **成功確認**
   ```bash
   adb shell dpm get-active-admin
   ```
   `uk.nktnet.webviewkiosk/.WebviewKioskAdminReceiver` が表示されていればOK

## アプリケーション設定

### Step 3: Webview Kioskの初期設定

#### 1. ホーム画面に設定

1. Webview Kiosk を起動
2. 設定 → 一般 → デフォルトランチャー
3. Webview Kiosk を選択
4. 以後、デバイス起動時に自動的にWebview Kioskが起動

#### 2. セキュリティ設定

1. 設定アイコン → セキュリティ セクション
2. **認証方法**: パスワード を選択
3. **パスワード**: 受付スタッフのみが知っているパスワードを設定（例：`reception2025`）
4. **確認**: パスワードをもう一度入力して確認

**パスワード忘れ対策：** 
- パスワードを紙に記録して、スタッフルームの鍵付きファイルボックスに保管してください

#### 3. ウェブコンテンツ設定

1. 設定アイコン → ウェブコンテンツ セクション
2. **ウェブサイト URL**: 
   ```
   https://www.e-shisetsu.e-aichi.jp/web_info.html
   ```
3. **ホワイトリスト（許可リスト）**:
   ```
   https://www\.e-shisetsu\.e-aichi\.jp/.*
   ```
   - この正規表現により、`https://www.e-shisetsu.e-aichi.jp/` 配下のすべてのページにアクセス可能
   - 追加サイトがある場合は、`|` で区切って複数指定可能
     - 例：`https://www\.e-shisetsu\.e-aichi\.jp/.*|https://example\.jp/.*`

4. **ブラックリスト（禁止リスト）**: 空白（ホワイトリスト方式を採用）

#### 4. タイムアウト設定

1. 設定アイコン → ウェブライフサイクル セクション
2. **不活動時にリセット (秒)**: `600`
   - 10分間（600秒）操作がない場合、自動的にリセット
   - ログイン情報や入力データもすべてクリア
   - ホーム ページに戻ります

3. **画面消灯時にリセット**: ON （推奨）

4. **不活動時に画面消灯 (秒)**: `300`
   - 5分間操作がない場合、画面を消灯

#### 5. ロック機能設定

1. 設定アイコン → 一般 セクション
2. **Lock Task モード**: ON
   - ホーム画面や他のアプリへのアクセスを遮断
   - ステータスバーの操作を制限

3. **戻るボタン動作**: 無効 （推奨）
   - ユーザーが誤って戻るボタンを押してアプリを閉じることを防止

4. **ウェイクロック**: ON
   - 画面スリープ後、リモートコマンドで画面を点灯させることが可能

## デバイス側の設定（Android 設定）

### Step 4: スクリーンロック設定

1. **デバイス設定** → **セキュリティ**
2. **スクリーンロック**: 図形パターン または パスワード を設定
   - 短い暗証番号（例：`1234`）を推奨
3. **ロック画面タイムアウト**: 5分
4. **デバイス管理者**: Webview Kiosk を有効化

### Step 5: その他のセキュリティ設定

1. **開発者向けオプション**: 無効（デフォルト）
2. **USB デバッグ**: 無効（デフォルト）
3. **不明なソース**: 無効（デフォルト）
4. **自動タイムゾーン**: ON
5. **自動日時**: ON

### パスワード変更手順

1. Webview Kiosk を起動
2. 設定 → セキュリティ
3. **パスワード変更** を選択
4. 現在のパスワードを入力
5. 新しいパスワードを入力（2回）
6. **保存** をタップ

**推奨頻度**: 月1回以上

### トラブルシューティング

#### 問題 1: 設定画面にアクセスできない

**原因**: パスワードを忘れた

**解決策**:
1. デバイスをリセット
2. 再度Webview Kioskをインストール
3. アプリの初期設定から開始

#### 問題 2: URLフィルタリングが機能しない

**原因**: 正規表現が不正な形式

**確認項目**:
- バックスラッシュ `\` でエスケープされているか
- `.` （ドット）が含まれる場合、`\.` とエスケープされているか
- 例）正：`https://www\.e-shisetsu\.e-aichi\.jp/.*`

#### 問題 3: 画面が勝手に消える

**原因**: スクリーンタイムアウト設定が短すぎる

**解決策**:
1. Android 設定 → ディスプレイ
2. スクリーンタイムアウト → 10分以上に変更

#### 問題 4: 電源ボタンを押すと他のアプリが起動する

**原因**: Lock Task モードが無効

**解決策**:
1. Webview Kiosk 設定 → 一般
2. **Lock Task モード**: ON に設定
3. アプリを再起動

## セキュリティのベストプラクティス

### ✅ 実施すべき対策

1. **強力なパスワード設定**
   - 8文字以上
   - 大文字・小文字・数字を混在
   - 例：`R3cepti0n@2025`

2. **定期的なパスワード変更**
   - 月1回以上

3. **デバイス管理者権限の保護**
   - 管理者パスワードは共有しない
   - 定期的に権限確認

4. **設定情報のバックアップ**
   - Webview Kiosk → 設定 → エクスポート
   - 設定をJSONで保存
   - 安全な場所に保管

5. **アプリのアップデート**
   - 定期的に最新版をインストール

### ❌ 避けるべき操作

1. デバイス所有者権限を削除
2. Lock Task モードを無効化
3. URLホワイトリストを削除
4. パスワード設定を解除
5. 他のアプリをインストール

## 高度な設定（オプション）

### MQTT リモート管理（企業向け）

複数の端末を一元管理したい場合：

1. **MQTTブローカーの準備**
   - EMQX、Mosquitto など

2. **Webview Kiosk の MQTT 設定**
   - 設定 → リモート管理 → MQTT
   - ブローカーアドレス: `mqtt.example.com`
   - ポート: `1883`

3. **リモートコマンド実行**
   - 設定の更新
   - 画面オン/オフ
   - ページの強制リロード
