# Webview Kiosk - キオスク設定ガイド

## 概要

このドキュメントは、GalaxyTab A9+など受付端末として、Webview Kioskを使用して、特定のウェブサイト（例：https://www.e-shisetsu.e-aichi.jp/web_info.html）への安全なアクセスを提供する方法を説明します。

## 必要な設定

### 1. デバイス所有者（Device Owner）モード

キオスク機能を完全に有効化するには、デバイス所有者モードの設定が必要です。

#### セットアップ方法

**初期セットアップ時:**

1. 工場出荷状態のデバイスで最初の起動を行います
2. 初期セットアップ画面で以下のコマンドを実行します：
   ```bash
   adb shell dpm set-device-owner uk.nktnet.webviewkiosk/.WebviewKioskAdminReceiver
   ```

**既存デバイスの場合:**

通常、デバイス所有者モードはリセット後にのみ設定できます。

### 2. アプリケーション設定

#### 基本設定

1. **アプリを開く** → 設定アイコンをタップ
2. **セキュリティ** セクション:
   - 認証方法: パスワードを選択
   - パスワード: 受付スタッフのみが知っているパスワードを設定

#### URLフィルタリング設定

1. **ウェブコンテンツ** セクション:
   - **ウェブサイト**: `https://www.e-shisetsu.e-aichi.jp/web_info.html`を入力
   - **ホワイトリスト**: `https://www\.e-shisetsu\.e-aichi\.jp/.*`（正規表現）
   - **ブラックリスト**: 空白（ホワイトリストのみを使用）

#### タイムアウト設定

1. **ウェブライフサイクル** セクション:
   - **不活動時にリセット (秒)**: 600（10分）
   - この設定により、10分間操作がない場合、ログイン情報がクリアされて最初のページに戻ります

#### 画面ロック設定

1. **ロック機能** セクション:
   - **ロック機能を有効**: ON
   - **ロック機能パスワード**: セキュリティパスワードと同じまたは異なるパスワード

### 3. スクリーン保護機能

#### 自動スクリーンロック

1. **デバイス設定** → **セキュリティ**
   - スクリーンロック: 図形 または パスワード に設定
   - ロック画面タイムアウト: 5分

#### 画面スリープ時の自動復帰

Webview Kioskの設定で以下を有効化：

1. **一般** セクション:
   - **ウェイクロック**: ON（画面スリープ時にコマンド受信で自動復帰）

### 4. 物理ボタンの無効化

#### 戻るボタン設定

- Webview Kioskの**一般**セクションで:
  - **戻るボタン動作**: 無効
  - この設定により、戻るボタンが何もしなくなります

#### 電源ボタンの管理

デバイス所有者モードでの管理:

```bash
# 電源ボタンの長押しを無効化（Androidデバイス設定による）
# リセットボタンなどの物理キーは、デバイス管理によって制御されます
```

### 5. キオスク（Lock Task）モード

#### Lock Taskモードの有効化

1. **一般** セクション:
   - **Lock Taskモード**: ON
   - アプリケーション外への操作を防止します

#### 設定手順

1. Webview Kioskの設定で、Lock Taskを有効化
2. Lock Taskが実行中は:
   - ホーム画面へのアクセスが遮断
   - 他のアプリへのアクセスが遮断
   - ステータスバーの操作が制限

## 運用ガイドライン

### 通常の使用フロー

1. **スタートアップ**: タブレット起動時、自動的にWebview Kioskが起動
2. **サイトアクセス**: https://www.e-shisetsu.e-aichi.jp/web_info.html が自動で読み込まれます
3. **ユーザー操作**: 指定されたサイト内でのみ操作可能
4. **タイムアウト**: 10分間操作がない場合、セッション自動クリア

### パスワード変更

1. Webview Kioskの設定画面にアクセス（生体認証またはパスワード入力）
2. **セキュリティ** → パスワードを変更

### トラブルシューティング

#### 設定画面にアクセスできない

- パスワードを忘れた場合は、デバイスをリセット後、再度セットアップが必要

#### URLフィルタリングが機能しない

- ホワイトリスト正規表現を確認
- 以下の形式で確認: `https://www\.e-shisetsu\.e-aichi\.jp/.*`

#### スクリーンが勝手に消える

- **デバイス設定** → **ディスプレイ** → スクリーンタイムアウト時間を確認
- ウェイクロック設定を確認

## セキュリティ推奨事項

1. **定期的なパスワード変更**: 月1回以上
2. **デバイス管理**: デバイス所有者権限を適切に管理
3. **アップデート**: Webview Kioskを常に最新版に保つ
4. **ネットワークセキュリティ**: 公開WiFiではなく、固定の有線接続推奨
5. **バックアップ**: 設定をエクスポート機能でバックアップ

## 詳細設定（オプション）

### MQTT統合（リモート管理）

1. MQTTブローカーを設定
2. トピックを`webview/kiosk/commands`に設定
3. リモートで設定を管理可能

### UnifiedPush（分散プッシュ通知）

- さらなるセキュリティとプライバシーを求める場合に使用
